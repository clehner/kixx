Quick and Dirty Content Management System

Overview

Our website runs on the Google App Engine (GAE) platform. It is written in Python and uses a quick and dirty content management system with Django templates. Communication between this toolpack and the GAE host is done with a RESTful api using HTTP + JSON.

Use this list of links to see the raw api:
  • http://www.fireworksproject.com/content-manager/configs
  • http://www.fireworksproject.com/content-manager/templates/
  • http://www.fireworksproject.com/content-manager/inventory/
  • http://www.fireworksproject.com/content-manager/pages/

The Inventory Tab

The inventory tab will be the most used area of this toolpack. It provides a list of our complete content inventory for the whole website. More importantly, each content item on every page can be changed, edited, and updated using the content inventory tab. For the time being, the only way to identify where a content inventory item appears on the live website is to look at the description of it, so we need to make sure our descriptions are accurate when we post new inventory items.

Configuration Tab

The configuration tab provides a list of all the currently live pages and provides a form to edit the JSON configurations of each page. You should not edit any of the configurations unless you really know what you are doing.

Templates Tab

The templates tab simply provides a list of all the currently live templates for reference.
